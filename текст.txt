Имеется файл с исходными данными, каждая строка которого - филиал Банка и его характеристики (есть суммарные строки, которые нужно игнорировать)

1. Создать таблицы БД со структурой, позволяющей хранить 
а) Наименование, номер (берется из наименования), тип подразделения, адрес и географические координаты филиала; 
б) расписание работы филиала с возможностью агрегации времени работы за период; 
в) актуальность расписания работы филиала (если в какой-то момент времени расписание меняется, то для более ранних дат должно действовать прежнее расписание, для поздних - новое)

Требования к БД:
а) СУБД - Microsoft SQL Server/Oracle/PostgreSQL
б) Изначально БД должна быть пустой. Приложить SQL-скрипт для создания необходимых таблиц, ключей и индексов.


2. Написать запросы для выборки

- Кросс-запрос, в заголовках строк - населенный пункт (из адреса), в заголовках столбцов - тип подразделения. Значения в ячейках - кол-во филиалов. Допустимо создать условно статичный SQL-запрос
- Задан период между днем А и днем Б включительно. Для каждого филиала требуется вывести долю рабочего времени, которую отработал филиал в течение периода от А до Б по сравнению со временем работы в течение месяца. Если период А-Б протекает в двух и более месяцах, то необходимо дополнительно отобразить группировку по месяцам.

Требования к запросам:
  а) Переменные запроса должны редактироваться только в одном месте скрипта.
  б) Допустимо создавать и применять вспомогательные хранимые процедуры и функции.


3. Написать модуль Python с использованием Flask 

Модуль должен возвращать список отделений в указанном городе в формате json.

Пример api-запроса: http://127.0.0.1:8080/api/list/Ангарск

Пример json:

{
  "response": {
    "api": "v1", 
    "page": "list", 
    "values": [
      {
        "id": "1", 
        "code": "01885860196", 
        "name": "Доп.офис №8586/0196", 
        "address": "665832, Иркутская область, г.Ангарск, ул.7 микрорайон, дом 25"
      }, 
      ... 
      {
        "id": "1", 
        "code": "01885860201", 
        "name": "Доп.офис №8586/0201", 
        "address": "665830, Иркутская область, г.Ангарск, 60 квартал, дом 5/6, помещение 22"
      }
    ]
  }
}



Примечания к исходным данным:
1. Филиал идентифицируется двумя числами, записанными в названии через "/". Например, "Доп.офис №8586/0133".
2. Время работы филиала задано двумя текстовыми колонками: "Общий режим работы/Дни" (через запятую указаны дни недели, когда работает филиал)  и "Общий режим работы/Время работы" (каждому дню соответствует пара часовых строк - время открытия и закрытия - плюс опционально часы перерыва в работе филиала для каждого дня (перерыв указан обязательно в скобках)

Например, "Вт.,Ср." + "09:00 19:00 09:00 17:00" означает, что во вторник филиал работает с 09:00 до 19:00, в среду - с 09:00 до 17:00.   Еще пример: "Чт.,Сб." + "08:00 17:00 (13:00 14:00) 10:00 14:00"  филиал работает в четверг с 08:00 до 17:00, обед с 13:00 до 14:00, и в субботу с 10:00 до 14:00.
